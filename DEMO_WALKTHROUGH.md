## 2. Архитектура (3 мин)

### Что показываем
1. **Mermaid-диаграмму** в README — общая архитектура
2. **Docker Desktop** — 4 работающих контейнера (PostgreSQL, Redis, ChromaDB, Neo4j)
3. **Terminal** — `curl http://localhost:8080/info` для демо API Gateway
4. **Mermaid-диаграмму** LangGraph workflow — 5 узлов

### Что говорим
> "Архитектура микросервисная — 5 независимых сервисов за API Gateway на FastAPI.
> Данные хранятся в 4 базах: ChromaDB для векторов, PostgreSQL для метаданных,
> Neo4j для графа юридических определений, Redis для кэша.
>
> Ключевая инновация — LangGraph CRAG workflow. Это не просто 'запрос ответ'.
> Система сначала ищет документы гибридным поиском, потом LLM оценивает их
> релевантность, если качество низкое — автоматически переформулирует запрос
> и ищет заново, генерирует ответ и САМА проверяет себя на галлюцинации.
> Это agentic подход — система принимает решения на каждом шаге."

### Команды для демо
```bash
# Показать здоровье системы
curl http://localhost:8080/health/all

# Показать информацию о сервисах
curl http://localhost:8080/info

# Docker контейнеры
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

## 6. Бизнес-результаты (2 мин)

### Ключевые цифры
- **97.5% точность** — 39 из 40 правильных ответов на юридическом бенчмарке

- **5500+ клиентов** компании можно обслуживать автоматизированно
